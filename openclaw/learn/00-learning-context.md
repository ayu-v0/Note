# 00. 学习协作约定

这份文件记录面向新手的协作偏好，后续整理 OpenClaw 项目知识时优先放在 `.workplace/learn/` 下。

## 使用者背景

- 使用者是项目新手。
- 解释项目知识时，先给心智模型，再给源码入口。
- 避免只列结论；需要说明为什么从这些文件开始看。
- 复杂主题拆成小步骤，每一步配 repo-root 路径和可运行的搜索命令。

## 知识沉淀位置

新增或整理 OpenClaw 项目知识时，优先写入 `.workplace/learn/`。

建议格式：

- 一个主题一篇 Markdown。
- 文件名使用两位数字前缀，保持阅读顺序清晰。
- 每篇先说明“这篇解决什么问题”，再列核心概念、源码入口、阅读顺序和练习命令。
- 路径只写 repo-root 相对路径，例如 `src/gateway/server.ts`。

## 后续补充原则

- 新手优先：先解释项目结构、运行链路、边界和常见术语。
- 源码优先：结论需要能对应到仓库里的文档或源码。
- 小步验证：能用 `rg`、targeted tests 或文档脚本验证的内容，要给出验证方式。
- 不把不确定推断写成事实；不确定时标注待验证入口。
